# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a **Sui CTF (Capture the Flag)** repository for learning Sui Move smart contracts, the Sui object model, and Programmable Transaction Blocks (PTBs). The CTF is deployed on **Sui testnet**.

- **Deployed contract:** `0xd56e5075ba297f9e37085a37bb0abba69fabdf9987f8f4a6086a3693d88efbfd`
- **Network:** Sui testnet

## Commands

All script commands are run from the `scripts/` directory using `pnpm`:

```bash
cd scripts
pnpm install           # Install dependencies

pnpm init-keypair      # Generate keypair → scripts/keypair.json
pnpm moving-window     # Run moving_window challenge
pnpm merchant          # Run merchant challenge
pnpm lootboxes         # Run lootboxes challenge
pnpm staking           # Run staking challenge
pnpm sabotage-arena    # Run sabotage_arena challenge
```

To build/deploy Move contracts (from the contract directory):
```bash
sui move build --path contracts/
sui client publish contracts/ --gas-budget 100000000

# For the exploit contract:
sui move build --path exploit/
sui client publish exploit/ --gas-budget 100000000
```

## Architecture

### Two main components:

**1. `contracts/`** — Sui Move smart contracts (package: `ctf`)
- `sources/flag.move` — Base `Flag` object used by all challenges; `ctf::flag::new()` is package-private
- `sources/moving_window.move` — Time-based challenge using `sui::clock`
- `sources/merchant.move` — Purchase a flag with USDC tokens
- `sources/lootboxes.move` — 1-in-100,000 random chance; exploitable via composition attack on `public` randomness
- `sources/staking.move` — Stake ≥1 SUI for ≥1 week to earn a flag
- `sources/sabotage_arena.move` — PvP; build shield to 12, claim flag (10-min cooldown per action)

**2. `scripts/src/`** — TypeScript solve scripts (package: `sui-ctf-scripts`)
- Uses `@mysten/sui` SDK with `SuiGrpcClient` (not the older `SuiClient`)
- Keypair loaded from `scripts/keypair.json` (generated by `pnpm init-keypair`)
- Each challenge has a corresponding `.ts` file matching its Move module name

**3. `exploit/`** — Custom Move package for the lootboxes challenge
- Depends on `ctf = { local = "../contracts" }` with the deployed address hardcoded
- The exploit pattern: call `lootboxes::open_lootbox()` then `lootboxes::extract_flag()` — if no flag, `extract_flag` aborts the entire tx, rolling back the USDC payment

### Key constants used across scripts:
- `CTF_PACKAGE_ID` = `0xd56e5075ba297f9e37085a37bb0abba69fabdf9987f8f4a6086a3693d88efbfd`
- `CLOCK_ID` = `0x6` (Sui shared Clock object)
- `RANDOM_ID` = `0x8` (Sui shared Random object)
- USDC package: `0xa1ec7fc00a6f40db9693ad1415d0c193ad3906494428cf252621037bd7117e29`

### Lootboxes exploit design:
The lootboxes challenge uses `#[allow(lint(public_random))]` on a `public` function, making it vulnerable to composition attacks. The `exploit/` package deploys a single `entry fun exploit()` that wraps the lootbox call — if the random draw fails, `extract_flag` aborts the transaction so no USDC is spent. `lootboxes.ts` loops this until success.
